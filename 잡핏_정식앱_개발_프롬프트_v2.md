# 잡핏(JobFit) 정식 앱 개발 프롬프트 v2

> 이 프롬프트를 Claude에게 전달하면 잡핏 알바생 앱 + 사장님 앱을 Firebase 연동 정식 앱으로 개발할 수 있습니다.
> 프롬프트가 길므로 섹션별로 나눠서 전달해도 됩니다.

당신은 단기 알바 알선 플랫폼 "잡핏(JobFit)"의 풀스택 개발자입니다.
아래 기획을 기반으로 알바생 앱(jobfit-app.html)과 사장님 앱(jobfit-boss.html)을 정식 서비스 수준으로 개발해주세요.
Firebase Firestore를 백엔드로 사용하며, 카카오 로그인/카카오맵/FCM 푸시를 실연동합니다.

[4-1. 앱 시작 시 체크 — 강제 업데이트 / 점검 모드] ★ 신규

앱 로드 시 가장 먼저 appConfig/general 문서를 조회:
① maintenanceMode === true → 점검 안내 전체 화면 표시 (maintenanceMessage), 다른 기능 차단
② 현재 앱 버전 < minAppVersion → 업데이트 필요 전체 화면 표시
   - "업데이트 하기" 버튼 → 앱스토어/플레이스토어 링크 (웹 버전은 새로고침)
   - 업데이트 전까지 다른 기능 차단
③ 정상 → 스플래시 → 로그인 플로우 진행

앱 버전은 HTML meta 태그 또는 JS 상수로 관리:
const APP_VERSION = "1.0.0";

[4-2. 인증 플로우 — 단일 세션 정책 포함] ★ 강화

스플래시 화면 (1.5초) → 카카오 로그인 화면
- 카카오 JS SDK로 로그인
- 카카오에서 받은 토큰으로 Firebase Auth Custom Token 발급 (Cloud Functions)
- Firestore users/{uid} 존재 여부 확인
  → 있으면: 홈으로 이동
  → 없으면: 회원가입 플로우 시작
- 로그인 실패 시 재시도 버튼 노출

★ 단일 세션 관리:
- 로그인 성공 시 고유 sessionId 생성 (crypto.randomUUID())
- users/{uid}.currentSessionId에 저장
- 앱 사용 중 주기적으로 (30초마다) currentSessionId 비교
  → 불일치 시 = 다른 기기에서 로그인됨
  → "다른 기기에서 로그인되었습니다" 알림 → 강제 로그아웃 → 로그인 화면
- 이렇게 하면 출근 버튼 중복 실행 방지

[4-3. 회원가입 (3단계 — 임시저장 없음)]

Step 1 — 기본정보 입력:
  - 이름 (필수)
  - 생년월일 (필수, date picker)
  - 휴대폰 (필수, 숫자만)
  - 성별 (필수, 라디오)
  - 지역 (필수, 서울 25개 구 + 경기도 시·군 select)

Step 2 — 급여 계좌 등록:
  - 은행 선택 (select)
  - 계좌번호 (숫자만)
  - 예금주 (텍스트)
  ※ 실명/계좌 인증 없음 — 직접 입력

Step 3 — 동의 및 서명:
  - 개인정보 활용 동의 내용 표시 + Canvas 서명 패드
  - 안전교육 확인 내용 표시 + Canvas 서명 패드
  - ★ 마케팅 알림 수신 동의 (선택, 체크박스)
    → "긴급 구인, 포인트 안내 등 마케팅 알림을 받으시겠습니까?"
    → users.marketingConsent에 저장
  - 서명 완료 → Firestore users/{uid} 문서 생성 → 홈 이동

[4-4. 홈 탭]

상단:
  - "잡핏" 로고 + 알림 아이콘 (미읽음 카운트 뱃지)

카드 영역:
  - 이번달 근무 횟수 카드
  - 포인트 현황 카드 + "출금하기" 버튼
    → 출금 조건: 30,000P 이상 시 10,000P 단위
  - ★ 경고 현황 표시 (warningCount > 0 일 때)
    → "⚠️ 경고 {n}/3회" 뱃지 (빨간색)
    → isBanned === true 일 때 상단 배너: "3회 경고로 신청이 제한되었습니다. 문의하기"

내 신청 현황 리스트 (날짜순, Firestore 실시간 리스너):
  - 각 항목: 공고명, 날짜, 시간, 상태 뱃지
  - 상태별 색상:
    승인대기(--color-warning) / 승인완료(--color-success) /
    거절(--color-danger) / 출근완료(--color-royal) / 근무종료(--color-sub-text)
  - 승인완료 + 출근 60분 전이면 "출근하기" 버튼 표시
  - 승인대기 + 시작 2시간 전까지 "신청취소" 버튼 표시

[4-5. 알바 찾기 탭]

상단 필터 바:
  - 날짜 / 지역 / 업종 필터
  - 리스트 뷰 / 지도 뷰 전환 토글

리스트 뷰:
  - jobs 컬렉션 status="open" 쿼리
  - 긴급 공고 최상단 고정 + "긴급" 뱃지
  - 각 카드: 공고명, 거점명, 날짜, 시간, 시급, 추가포인트, 모집/승인 인원
  - 마감 숨김, 찜 하트 아이콘

지도 뷰:
  - 카카오맵 마커 + 인포윈도우 → 상세 이동

공고 상세:
  - 전체 정보 + 카카오맵 + 이동수단(통근버스 시간표)
  - "신청하기" 버튼
    → ★ isBanned === true 시 신청 차단 + "3회 경고로 신청이 제한되었습니다" 안내
    → 시간 중복 체크 (겹치면 차단)
    → 성공 → applications 생성 (autoRejectAt: +12시간)

[4-9. 출근 시스템 — GPS 2단계 검증] ★ 핵심 변경

기존과 달리 출근 버튼 + GPS 검증 2단계로 동작:

① "출근하기" 버튼 클릭 (근무 60분 전부터 활성화)
② 확인 팝업 → 최종 확인 (잘못 누름 방지)
③ 근로계약서 자동 생성 → Canvas 서명
④ 서명 완료 시:
   - contracts 문서 생성
   - attendance.status = "checked_in_no_gps" (출근 신청 완료, GPS 미확인)
   - attendance.checkInTime = now
   - 알바생 화면: "거점 근처에 도착하면 자동으로 출근이 확인됩니다" 안내 표시

⑤ GPS 위치 추적 시작 (Geolocation API watchPosition):
   - 해당 거점 좌표와 현재 위치 간 거리 계산 (Haversine 공식)
   - 500m 이내 진입 시:
     → attendance.status = "checked_in"
     → attendance.gpsVerified = true
     → attendance.gpsVerifiedTime = now
     → 사장님 앱 출근 명단에 체크 표시 (실시간 반영)
     → 알바생에게 "출근이 확인되었습니다!" 토스트
     → watchPosition 중지
   - 500m 초과 상태 유지 시:
     → 화면에 "현재 거점까지 약 {거리}m" 실시간 표시
     → 시작시간 +30분까지 500m 미진입 → 자동 결근 처리

거리 계산 함수:
function getDistance(lat1, lng1, lat2, lng2) {
  const R = 6371000; // 지구 반경 (m)
  const dLat = (lat2 - lat1) * Math.PI / 180;
  const dLng = (lng2 - lng1) * Math.PI / 180;
  const a = Math.sin(dLat/2)**2 +
            Math.cos(lat1*Math.PI/180) * Math.cos(lat2*Math.PI/180) *
            Math.sin(dLng/2)**2;
  return R * 2 * Math.atan2(Math.sqrt(a), Math.sqrt(1-a));
}

경고 단계:

- 2회 경고: 경고 알림 + 홈 화면 "⚠️ 경고 2/3회" + "다음 경고 시 신청이 제한됩니다" 안내
- 3회 경고: users.isBanned = true → 모든 공고 신청 차단
  → "3회 경고로 신청이 제한되었습니다. 문의하기를 통해 해제를 요청할 수 있습니다."
  → 사장님이 직접 warningCount 초기화 + isBanned = false로 해제 가능

처리 주체:
- 자동 결근 시 시스템이 warningCount +1
- 사장님/매니저가 수동으로 경고 부여/해제 가능 (근무자 탭에서)

exports.scheduledFirestoreBackup = functions.pubsub
  .schedule('0 3 * * *')          // 매일 03:00 KST
  .timeZone('Asia/Seoul')
  .onRun(async () => {
    const client = new firestore.v1.FirestoreAdminClient();
    const projectId = process.env.GCP_PROJECT;
    const databaseName = client.databasePath(projectId, '(default)');
    const bucket = 'gs://jobfit-backup';

      outputUriPrefix: `${bucket}/${timestamp}`,
      collectionIds: [] // 전체 컬렉션
